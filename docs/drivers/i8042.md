# i8042 Controller Driver

The i8042 controller is a lower-kernel hardware mechanism. Supervisor policy and userspace services do not access ports `0x60` or `0x64` directly.

## Timeout policy

All i8042 waits are bounded. Waiting for input-buffer-empty or output-buffer-full returns an error on timeout instead of panicking or spinning forever. Flush operations use a maximum read count.

## Status/data behavior

- Status port `0x64` is read before data port `0x60`.
- Data port `0x60` is read only when output-buffer-full is set.
- Timeout and parity status bits are reported as controller errors.
- AUX/mouse bytes are not decoded as keyboard scancodes.

## Boot policy

A controller timeout degrades or disables the input device. It must not stop rootfs, Supervisor, MTSS, or PID1 handoff.
